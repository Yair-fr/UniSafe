# Contributing to UniSafe

Thank you for your interest in contributing to **UniSafe**! We welcome contributions of all kinds — from bug fixes to new features.

## 🚀 Getting Started

1. **Fork** the repository
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/UniSafe.git
   cd UniSafe
   ```
3. **Create a branch** for your feature:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Set up the development environment** — follow the [Quick Start](README.md#-quick-start) instructions
5. **Make your changes** and test them locally

## 📝 Commit Guidelines

We follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add new property filter by district
fix: correct Arnona calculation for Haifa Zone B
docs: update API reference for /api/evaluate
test: add unit tests for CBS CPI client
```

## 🔀 Pull Request Process

1. Ensure your code follows the existing style patterns
2. Update the README if you've added new API endpoints or features
3. Test your changes locally by running the server and verifying endpoints
4. Submit a Pull Request with a clear description of your changes

## 🐛 Reporting Bugs

Open an [Issue](https://github.com/Yair-fr/unisafe/issues) with:

- Steps to reproduce the bug
- Expected vs actual behavior
- Your OS and Python version
- Relevant error logs

## 💡 Feature Requests

We welcome feature ideas! Open an [Issue](https://github.com/Yair-fr/unisafe/issues) and label it as `enhancement`.
